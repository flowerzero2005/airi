# 语音输出优化 - 测试指南

## 📋 优化内容

本次优化解决了 AI 语音回复分段过于频繁、停顿不自然的问题。

### 问题描述
- **优化前**：每 2-8 个词就切断一次，导致 100 词的回答被切成 12-50 个片段
- **优化后**：每 30-100 个词切断，100 词的回答只需 2-5 个片段

## 🚀 如何测试

### 1. 启动应用
```bash
pnpm dev:tamagotchi
```

### 2. 配置语音输出
1. 打开应用后，点击右上角的设置图标
2. 导航到：**设置 → 系统 → 语音输出**
3. 选择输出模式：
   - **快速响应**：立即播放，但可能有停顿（原始行为）
   - **平衡模式**（推荐）：等待完整句子组，平衡速度和流畅度
   - **流畅模式**：等待完整段落，最流畅但延迟较高
   - **自定义**：手动调整参数

### 3. 测试场景

#### 场景 A：短回答（20-50 词）
**测试问题**：
```
"请简单介绍一下你自己"
```

**预期效果**：
- 平衡模式：1 个完整片段
- 快速模式：3-6 个片段

#### 场景 B：中等回答（100-200 词）
**测试问题**：
```
"请详细解释一下什么是机器学习，包括它的基本原理和应用场景"
```

**预期效果**：
- 平衡模式：2-4 个片段，每个片段是完整的段落或句子组
- 快速模式：12-25 个片段，频繁停顿

#### 场景 C：长回答（300+ 词）
**测试问题**：
```
"请写一个关于人工智能发展历史的短文，包括重要的里程碑事件"
```

**预期效果**：
- 流畅模式：2-3 个大段落，非常流畅
- 平衡模式：4-6 个片段，较流畅
- 快速模式：30-50 个片段，体验较差

## 📊 性能指标

| 模式 | 首次响应延迟 | 分段数量 | 流畅度 | 适用场景 |
|------|------------|---------|--------|---------|
| 快速响应 | 0.5-1s | 多 | ⭐⭐ | 需要快速反馈 |
| 平衡模式 | 1-2s | 中 | ⭐⭐⭐⭐ | 日常使用（推荐） |
| 流畅模式 | 2-4s | 少 | ⭐⭐⭐⭐⭐ | 追求最佳体验 |

## 🔧 自定义参数说明

如果选择"自定义"模式，可以调整以下参数：

### 快速处理（Boost）
- **范围**：0-5
- **说明**：前 N 个片段快速处理，0 表示禁用
- **推荐值**：0（禁用）或 1（轻微加速）

### 最小词数（Minimum Words）
- **范围**：5-50
- **说明**：开始合成前需要累积的最少词数
- **推荐值**：20-30

### 最大词数（Maximum Words）
- **范围**：20-200
- **说明**：单个片段的最大词数限制
- **推荐值**：80-100

## 🐛 已知问题

1. **TTS 服务限制**：某些 TTS 服务可能有单次请求字符数限制（通常 1000-5000 字符）
2. **网络延迟**：首次响应延迟会受网络状况影响
3. **流式输出**：AI 是流式生成的，无法完全消除等待时间
