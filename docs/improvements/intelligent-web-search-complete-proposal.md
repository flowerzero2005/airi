# AIRI 智能联网系统 - 完整设计方案

## 项目概述

设计一个**智能化、情境感知、人设一致**的联网系统，让 AIRI 能够：

1. **理解用户真实意图** - 而不是简单的关键词匹配
2. **智能判断查询时机** - 知道何时该查，何时不该查
3. **基于人设过滤信息** - 选择符合角色特点的内容
4. **自然表达知识** - "装不是很懂"，保持对话体验

## 关键设计原则

### 1. 智能意图识别

**不是简单的关键词匹配**：
- ❌ "最近" → 触发联网
- ✅ 理解用户是想闲聊还是真的需要信息

**考虑对话上下文**：
- 如果之前在聊游戏 → "有什么好玩的"可能指游戏
- 如果之前在聊电影 → "有什么好玩的"可能指电影

### 2. 情境感知查询

**何时查询**：
- 用户明确询问时效性信息
- 用户询问具体数据或事实
- 用户表现出强烈的信息需求

**何时不查询**：
- 用户只是闲聊或分享感受
- 话题可以通过对话澄清
- 查询会严重打断对话流畅度

**何时延迟查询**：
- 问题模糊，需要先澄清
- 可以后台查询，不阻塞对话

### 3. 基于人设过滤 ⭐

**二次元女孩人设的特点**：
- **超级喜欢**：动漫、游戏、梗、可爱的东西
- **比较喜欢**：音乐、美食、艺术（二次元相关）
- **不太感兴趣**：政治、经济、深奥理论

**过滤策略**：
```typescript
// 示例：查询"最近新闻"
原始结果: [
  '国际政治动态',
  '经济数据报告',
  '新番动画发布',
  '游戏展会消息',
  '科技产品发布'
]

人设过滤后: [
  '新番动画发布' (0.95), // 超高相关
  '游戏展会消息' (0.82), // 高相关
  '科技产品发布' (0.45) // 中等相关（作为AI有基础兴趣）
]

被过滤: [
  '国际政治动态' (0.10), // 不符合兴趣
  '经济数据报告' (0.05) // 不符合兴趣
]
```

### 4. 社交智能表达

**"装不是很懂"的艺术**：
```typescript
// 场景：用户问专业技术问题
用户: "Python的GIL是什么？"

// ❌ 错误（过度展示专业知识）
AI: "GIL（Global Interpreter Lock）是Python解释器的全局锁，
     它确保同一时间只有一个线程执行Python字节码..."

// ✅ 正确（符合人设 + 装不太懂）
AI: "嗯...GIL好像是Python里的一个锁？"
AI: "我记得它会影响多线程性能，但具体怎么工作的我不太清楚诶"
AI: "你是遇到多线程的问题了吗？"
[转移话题到用户需求，而不是展示知识]
```

**渐进式信息披露**：
```typescript
// 不要一次性倾倒所有信息
❌ 错误:
"Python 3.12有以下新特性：1. 性能提升15% 2. 改进的错误消息
 3. 新的类型系统特性 4. f-string改进 5. ..."

✅ 正确:
第一轮: "Python 3.12好像更新了不少东西诶"
第二轮: "最明显的应该是速度变快了，还有报错提示变友好了"
第三轮: [如果用户想了解更多] "你想了解哪方面的改进呀？"
```

3. **响应延迟** - 目标 <3秒

### 社交智能指标
1. **过度展示知识次数** - 目标 <5%
2. **打断对话次数** - 目标 <10%

## 风险与挑战

### 技术风险
1. **意图理解偏差** - 缓解：多轮验证 + 规则引擎
2. **人设过滤过度** - 缓解：可调节的过滤阈值
3. **响应延迟** - 缓解：后台查询 + 缓存

### 产品风险
1. **过度谨慎不查询** - 缓解：可调节的保守度
2. **人设限制信息** - 缓解：保留"专业模式"选项
3. **"装不懂"过度** - 缓解：根据用户偏好调整
