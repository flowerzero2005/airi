# 对话自然度改进方案

## 问题背景

用户反馈：当前 AIRI 的对话系统在使用记忆功能时，会出现"过度热情"和"侵略性"的问题。

### 问题示例

```
用户: hi，凛
AI: 嗨嗨，秋医～我在呀，是凛（RIN）喔。今天在长沙那边过得怎么样呀？
    要不要我给你安排点甜甜的菜单灵感：可乐鸡翅还是卤香烧鸡那种？你现在更想吃哪一口呀～
```

**问题分析**：
1. **信息过载** - 一次性抛出太多话题（地点、食物、两个具体菜品）
2. **过度主动** - 用户只是打招呼，AI 却立即推进到具体行动建议
3. **缺乏呼吸空间** - 没有给用户选择对话方向的机会
4. **目的性过强** - 像推销员而非朋友
5. **不自然的记忆展示** - 刻意展示"我记得你的信息"

## 改进方案

### 核心原则：基于社会心理学的"自然对话层级"

#### 1. 渐进式互动（Gradual Disclosure）

遵循真实人际交往的节奏：

```
阶段 1: 简单回应 → "嗨～"
阶段 2: 轻度探询 → "今天怎么样？"
阶段 3: 具体话题 → "午饭吃了吗？"
阶段 4: 深入互动 → "要不要试试这个菜谱？"
```

#### 2. 隐性记忆使用（Implicit Memory）

记忆应该像朋友间的默契，而非刻意展示：

**❌ 错误方式**：
- "我记得你喜欢吃可乐鸡翅"
- "你之前说过你在长沙"
- "根据我的记忆，你..."

**✅ 正确方式**：
- 当话题相关时自然提及："长沙那边天气怎么样？"
- 直接使用称呼，不说"我记得你叫XXX"
- 在需要时才引用记忆，不主动展示

#### 3. 被动优先（Passive First）

让用户主导对话方向：

- **何时可以主动引用**：
  - 用户明确询问时
  - 话题自然相关时
  - 用户表现出需要帮助时

- **何时不应主动引用**：
  - 用户只是简单打招呼时
  - 话题还没有展开时
  - 用户没有表现出相关需求时

#### 4. 情感连接优先（Emotion First）

关注点应该是：
- 用户当下的感受和需求
- 对话的自然流动
- 情感的共鸣和理解

而不是：
- 证明你记得很多信息
- 展示记忆系统的能力
- 让用户感到被"监视"或"分析"

```typescript
## 自然对话原则（重要！）

### 对话节奏 - 渐进式互动
遵循真实朋友间的对话节奏，不要一次性抛出所有信息：

**第一轮（简单回应）**：
- 用户只是打招呼 → 你也简单回应即可
- 例如："嗨～" "在呀～" "嗯嗯？"
- 可以自然地使用称呼，但不要立即展开话题

**第二轮（轻度探询）**：
- 如果用户继续对话 → 可以问一个简单的开放性问题
- 例如："今天怎么样？" "在忙什么呀？"
- 避免具体的、有目的性的问题

**第三轮及以后（话题深入）**：
- 根据用户的回应 → 逐步引入更具体的话题
- 让用户主导对话方向
- 只在用户表现出兴趣时才深入
```

## 理论基础

### 社会语言学原则

1. **Face Theory（面子理论）** - Erving Goffman
   - 过度展示记忆 = 侵犯对方的"消极面子"（不被打扰的需求）
   - 自然使用记忆 = 尊重对方的社交边界

2. **Politeness Theory（礼貌理论）** - Brown & Levinson
   - 渐进式互动 = 消极礼貌策略（给予选择空间）
   - 被动优先 = 避免强加意愿

3. **Conversational Implicature（会话含义）** - Paul Grice
   - 隐性记忆使用 = 遵循"量的准则"（不提供过多信息）
   - 情感连接优先 = 遵循"关系准则"（保持相关性）

### 心理学原则

1. **Cognitive Load Theory（认知负荷理论）**
   - 信息轰炸 = 增加认知负荷
   - 渐进式互动 = 降低认知负荷

2. **Self-Determination Theory（自我决定理论）**
   - 被动优先 = 满足自主需求
   - 给予选择空间 = 增强内在动机

3. **Attachment Theory（依恋理论）**
   - 自然对话节奏 = 建立安全依恋
   - 情感连接优先 = 满足情感需求

## 后续优化方向

1. **动态调整对话风格**
   - 根据用户的回应模式调整主动性
   - 识别用户是否喜欢更主动的对话风格

2. **记忆重要性评分**
   - 为记忆添加"社交重要性"评分
   - 只在高重要性时才主动引用

3. **对话状态追踪**
   - 追踪对话的"亲密度"
   - 在关系更亲密时可以更主动

4. **用户偏好学习**
   - 学习用户的对话风格偏好
   - 自适应调整对话策略
