# 🧠 智能记忆系统

## 概述

Airi 的智能记忆系统是一个完整的长短期记忆解决方案，让 AI 能够记住与用户的对话历史、个人偏好和重要信息，实现真正有记忆的智能交互体验。

## ✨ 核心特性

### 🎯 混合记忆提取策略

系统采用多层次的记忆提取机制，自动识别对话中的重要信息：

- **关键词匹配** - 识别姓名、地点、时间等关键信息
- **情感分析** - 捕捉用户的喜好、厌恶和情绪状态
- **上下文理解** - 基于对话上下文判断信息重要性
- **智能去重** - 自动合并相似记忆，避免冗余存储

### 🔍 智能记忆检索

#### 主动搜索（AI Tool）
AI 可以在对话中主动调用 `search_memory` 工具搜索相关记忆：

```typescript
// AI 可以这样使用
search_memory({
  query: '用户的名字',
  limit: 5
})
```

**特点：**
- AI 自主决定何时需要搜索记忆
- 不会刻意引导话题
- 自然融入对话流程

#### 被动注入（上下文感知）
系统会在每次对话前自动注入相关记忆：

- **长消息** - 基于用户当前问题搜索相关记忆
- **短消息** - 当用户消息少于10个字符时（如"嗯"、"好的"），基于最近3轮对话上下文搜索
- **智能关键词映射** - 将问题模式映射到搜索概念

```typescript
// 示例：用户问"你知道我是谁吗"
// 系统自动映射为搜索: ["用户", "姓名", "名字", "身份", "自称"]
```

### 📊 记忆分级系统

记忆按重要性分为三个等级：

| 等级 | 图标 | 说明 | 存储类型 |
|------|------|------|----------|
| 高优先级 | ⭐ | 用户个人信息、重要偏好 | Focus（焦点） |
| 中优先级 | 📌 | 一般性信息、话题讨论 | Note（笔记） |
| 低优先级 | 💡 | 临时信息 | 不存储 |

### 🗂️ 短期记忆管理

短期记忆即对话历史，支持灵活的管理策略：

#### 自动清理
- **阈值触发** - 当消息数量超过设定阈值时自动清理
- **保留策略** - 保留最近 N 条消息，删除更早的历史
- **定期检查** - 每5秒检查一次是否需要清理

#### 手动管理
- 查看当前对话的所有消息
- 一键清空短期记忆
- 实时统计消息数量

### 💾 持久化存储

#### 实时保存机制
- **用户消息** - 发送后立即保存到 IndexedDB
- **AI 回复** - 流式输出时每100个字符保存一次
- **完整保存** - 对话结束后完整保存
- **防丢失** - 即使快速刷新页面也不会丢失消息

#### 数据同步
- 本地优先策略（Local-First）
- 登录后自动同步到云端
- 支持多设备数据同步

### 🎨 记忆可视化

#### 长期记忆页面
- **统计卡片** - 总记忆数、重要记忆数、普通记忆数
- **搜索过滤** - 按内容、标签搜索记忆
- **重要性筛选** - 快速查看不同优先级的记忆
- **标签系统** - 自动提取和管理记忆标签
- **编辑功能** - 支持编辑和删除记忆

#### 短期记忆页面
- **消息列表** - 查看当前会话的所有消息
- **自动清理配置** - 设置阈值和保留数量
- **实时统计** - 显示当前消息数量
- **一键清理** - 手动清空对话历史

### 关键算法

#### 相似度计算
使用 Jaccard 相似度算法检测重复记忆：

```typescript
function calculateSimilarity(text1: string, text2: string): number {
  const words1 = new Set(text1.split(/\s+/))
  const words2 = new Set(text2.split(/\s+/))

  const intersection = new Set([...words1].filter(x => words2.has(x)))
  const union = new Set([...words1, ...words2])

  return intersection.size / union.size
}
```

## 🚀 使用示例

### 场景1：记住用户信息

**用户：** 我叫张三，是一名程序员

**系统行为：**
1. 提取关键信息："用户名：张三"、"职业：程序员"
2. 标记为高优先级记忆（⭐）
3. 保存到 Focus 分区

**下次对话：**

**用户：** 你知道我是谁吗？

**AI：** 你是张三，一名程序员！

### 场景2：上下文感知搜索

**对话历史：**
- 用户：我最近在学习 Vue.js
- AI：Vue.js 是一个很棒的前端框架...
- 用户：嗯

**系统行为：**
1. 检测到用户消息很短（"嗯"）
2. 基于最近对话上下文搜索："Vue.js"、"学习"、"前端"
3. 注入相关记忆到 AI 上下文

### 场景3：AI 主动搜索

**用户：** 我们上次聊到哪了？

**AI 行为：**
1. 调用 `search_memory({ query: "上次对话", limit: 5 })`
2. 获取最近的对话记忆
3. 总结并回复用户

## 🐛 故障排查

### 记忆未保存
1. 检查 Console 日志中的 `[MemoryManager]` 输出
2. 确认 Notebook Store 已加载
3. 检查 IndexedDB 存储空间

### 搜索无结果
1. 检查关键词映射配置
2. 查看记忆总数是否为0
3. 尝试使用更通用的搜索词

### 消息丢失
1. 检查持久化队列是否正常
2. 查看 Console 中的保存日志
3. 确认 IndexedDB 可用
